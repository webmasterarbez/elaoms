# Global Coding Style

## When to use this skill:

- When writing any new code in any programming language
- When naming variables, functions, classes, or files
- When refactoring code to be more readable or maintainable
- When reviewing code for style consistency
- When extracting common logic into reusable functions or modules
- When cleaning up dead code, unused imports, or commented-out blocks
- When splitting large functions into smaller, focused pieces
- When applying consistent indentation and formatting
- When following DRY (Don't Repeat Yourself) principles
- When deciding on backward compatibility approaches (default: skip unless required)
- When choosing between abbreviations and descriptive names (prefer descriptive)
