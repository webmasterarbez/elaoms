# Backend Migrations

## When to use this skill:

- When creating new database migration files
- When working on files in `/migrations/`, `/db/migrate/`, `/alembic/`, or `/prisma/migrations/` directories
- When adding, modifying, or removing database tables or columns
- When creating or modifying indexes on database tables
- When writing up/down or rollback migration methods
- When handling data migrations separate from schema changes
- When planning migration order for high-availability or zero-downtime deployments
- When reviewing migration files before deployment
- When working with Django migrations, Rails migrations, Alembic, Prisma migrate, Knex, or Flyway
- When needing to modify existing migration files (and understanding when this is safe)
